# 博客评论功能需求文档

## 1. 功能概述

为个人博客系统添加评论功能，允许访客对文章发表评论和回复。

## 2. 功能需求

### 2.1 评论发表

| 需求项 | 说明 |
|--------|------|
| 评论内容 | 支持纯文本评论，限制最大500字符 |
| 评论者身份 | 游客评论（填写昵称+邮箱） |
| 评论关联 | 评论必须关联到指定文章 |
| 评论时间 | 自动记录评论发表时间 |

### 2.2 评论回复

| 需求项 | 说明 |
|--------|------|
| 回复功能 | 支持对已有评论进行回复 |
| 嵌套层级 | 回复最多支持2层嵌套（评论→回复→回复的回复） |
| 回复标识 | 回复需显示被回复者的昵称 |

### 2.3 评论管理（后台管理）

| 需求项 | 说明 |
|--------|------|
| 查看评论 | 在后台管理中查看所有评论 |
| 删除评论 | 管理员可删除任意评论 |
| 删除子评论 | 删除父评论时，子评论一并删除 |

### 2.4 评论展示

| 需求项 | 说明 |
|--------|------|
| 评论列表 | 文章详情页底部显示评论列表 |
| 分页加载 | 评论较多时支持分页加载 |
| 时间格式 | 显示相对时间（如"3分钟前"、"2小时前"） |
| 回复展示 | 回复以嵌套形式展示在父评论下方 |

## 3. 数据模型

### 3.1 评论表 (t_comment)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 主键，自增 |
| article_id | BIGINT | 关联文章ID |
| parent_id | BIGINT | 父评论ID（NULL表示顶级评论） |
| reply_to_id | BIGINT | 回复目标评论ID |
| nickname | VARCHAR(50) | 评论者昵称 |
| email | VARCHAR(100) | 评论者邮箱 |
| content | VARCHAR(500) | 评论内容 |
| status | TINYINT | 状态：0待审核/1已发布/2已删除 |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |
| deleted | TINYINT | 逻辑删除标记 |

### 3.2 约束

- `article_id` 不能为空
- `content` 不能为空，最大500字符
- `nickname` 和 `email` 必填

## 4. 业务规则

### 4.1 评论层级规则

```
层级结构：
- 顶级评论 (parent_id = NULL, reply_to_id = NULL)
  - 一级回复 (parent_id = 顶级评论ID, reply_to_id = 顶级评论ID)
    - 二级回复 (parent_id = 顶级评论ID, reply_to_id = 一级回复ID)

规则：
- 顶级评论：直接评论文章
- 一级回复：回复顶级评论
- 二级回复：回复一级回复（不能再被回复）
```

### 4.2 数据验证规则

| 验证项 | 规则 |
|--------|------|
| 昵称 | 2-50字符 |
| 邮箱 | 有效邮箱格式 |
| 内容 | 1-500字符，不含HTML标签 |
| 文章ID | 必须存在且已发布 |

## 5. 非功能需求

### 5.1 性能要求

- 评论列表加载时间 < 500ms
- 评论发表响应时间 < 300ms

### 5.2 安全要求

- XSS防护：评论内容过滤HTML标签
- 邮箱脱敏：邮箱不在前端完整展示
